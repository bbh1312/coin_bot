# 관리자 페이지 업데이트 설계 메모

## 목표
- 전체 명령어 값을 웹에서 제어(기존 페이지 재사용)
- 사용자별 오픈 포지션 목록 확인
- 관리자 수동 진입(심볼 입력 → 사용자 선택 → 시장가 진입)
- 사용자별 자산/남은 진입금액 표시
- 사용자가 "관리자 셋팅 적용" 옵트인/아웃 가능(텔레그램 + status 표시)

## 요구사항 확정
- 포지션 목록: 롱/숏, 진입가, 수량, 엔진, ROI
- 관리자 수동 진입: 롱/숏 선택, 시장가 고정
- 물타기 규칙: 같은 방향 포지션 있으면 항상 추가 진입
- 포지션 모드: 무조건 헤지모드
- 엔진 라벨: MANUAL_ADMIN
- UI: 전체 선택 버튼 + 계정별 수동 진입 허용 토글
- 자산 표시: 선물지갑 USDT, 남은 진입금액(= entry_pct 기준 가용 USDT)

## UI 구성(초안)
1) 전역 설정 패널
   - 기존 웹 명령어 제어 유지
2) 사용자 리스트 패널
   - 계정명, 상태(활성/비활성), 수동 진입 허용 여부
   - 전체 선택/해제 버튼
3) 수동 진입 패널
   - 심볼 입력(예: PTB)
   - 롱/숏 선택
   - 대상 사용자 체크박스 리스트
   - 실행 버튼
4) 사용자별 포지션/자산 패널
   - 오픈 포지션 테이블
   - 선물지갑 USDT
   - 남은 진입금액(USDT)

## API/데이터 계약(초안)
- /status
  - 계정별: open_positions[], futures_usdt, entry_usdt_available
  - 계정별: admin_follow_enabled (관리자 셋팅 적용 여부)
- /admin/entry
  - input: symbol, side, account_ids[]
  - action: 시장가 진입 (MANUAL_ADMIN)
- /admin/accounts
  - 계정 목록 + 수동 진입 허용 토글 상태

## 텔레그램 명령(계정별)
- /admin_follow on|off
  - 사용자 옵트인/아웃
  - status 응답에 표시

## 정책
- 사용자 옵트인 OFF면 관리자 수동 진입 대상에서 제외
- 사용자 선택이 우선순위: 선택되었더라도 옵트인 OFF면 진입 금지
- 헤지모드 강제, 롱/숏 분리 포지션으로 처리
